# Safety and Responsible Use

This repository documents an academic cybersecurity lab project.

The project was performed only in a private, controlled, and authorized VMware lab environment.

## Purpose

The purpose of this repository is to present responsible cybersecurity learning, documentation, and portfolio work.

It does not provide executable attack tooling and should not be used for unauthorized testing.

## Allowed Use

- Reviewing cybersecurity lab documentation
- Learning about responsible security testing workflows
- Understanding structured reporting in security projects
- Reviewing safe portfolio-style cybersecurity documentation
- Demonstrating academic learning to recruiters or cybersecurity professionals

## Not Included

This repository does not include:

- Executable attack scripts
- Credentials
- Password files
- Real logs
- Raw scan outputs
- Sensitive lab artifacts
- Tools intended for unauthorized access

## Prohibited Use

Do not apply the concepts described here against:

- Public IP addresses
- Third-party systems
- Networks you do not own
- Accounts you do not have permission to test
- Production environments without written authorization

## Responsible Security Practice

Security testing must always be performed only with explicit authorization and within a clearly defined scope.

If you discover exposed SSH services, weak credentials, or any security issue in a real environment, report it responsibly to the system owner or security team.
